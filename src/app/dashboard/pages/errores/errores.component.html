GESTIONAR ERRORES DE SOLICITUDES

Hay dos formas en que una solicitud HTTP puede fallar:

  • Un error de red o de conexión puede impedir que la solicitud llegue al servidor backend.
  • El backend puede recibir la solicitud pero no puede procesarla y devolver una respuesta de error.

HttpClient captura ambos tipos de errores en HttpErrorResponse(Respuesta que representa un error o una falla ). Podemos inspeccionar la respuesta para identificar la causa del error y la acción adecuada para manejar el error.

Codigos de estado

  • Los errores de red o conexión tienen un código de estado de 0
  • Los errores de backend tienen el código de estado de error devuelto por el backend
